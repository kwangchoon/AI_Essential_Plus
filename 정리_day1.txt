= ONNX Export RuntimeError - _ModuleStackTracer proxy 추적 실패
ONNX export(torch.onnx.export) 실행 시 아래 에러 발생 RuntimeError: 8*s72 (140499551114480) is not tracked with proxy for <torch.fx.experimental.proxy_tensor._ModuleStackTracer object at 0x7fc89a133dd0>

예상 원인
실습 환경(conda sam) 셋업 누락 혹은 다른 실습서 버전 변경으로 인한 PyTorch 버전 불일치
수강생 환경: torch==2.4.0+cu121 (Dynamo ONNX Exporter 초기 버전, FX 트레이서 버그 존재)
실습 기준 환경: torch==2.10.0 (requirements2.txt 명시)
PyTorch 2.4.0에서 Dynamo exporter가 기본값(dynamo=True)으로 전환되었으나, _ModuleStackTracer가 일부 모델 패턴의 텐서 연산을 그래프에 기록하지 못하는 이슈 존재

[해결 방법]: 단계별 해결 순서 (1, 2, 3...)
방법 1 (권장 - 환경 일치) conda activate sam python -m pip install -r lecture/requirements2.txt → 설치 후 Jupyter 커널 재시작, 처음부터 순서대로 재실행
방법 2 (빠른 우회) torch.onnx.export() 호출 시 dynamo=False 옵션 추가 → 레거시 TorchScript 방식으로 export, 실습 결과에 영향 없음
개별 패키지만 업/다운그레이드하면 의존성 충돌 위험이 있으므로 requirements2.txt로 전체를 맞추는 것을 권장
버전 확인: import torch; print(torch.version)
[주의 사항]: 이 작업을 할 때 코치가 놓치지 말아야 할 포인트
파이토치의 버전 확인이 우선 되어야 함 .
